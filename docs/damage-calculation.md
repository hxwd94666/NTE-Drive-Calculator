# 伤害计算规则（项目金标准）

本文件记录项目当前确认的伤害计算规则。游戏官方静态数据负责提供技能倍率、等级曲线、伤害
属性、固定暴击率、敌方属性和环合常量；本规则定义这些输入如何进入各乘区。公式以用户确认的
项目金标准为准，官方文件用于补齐可追溯的数值，不以未确认的运行逻辑反向改写项目规则。

Python API 的百分比使用小数，例如 `20%` 写为 `0.20`。

## 数据可信度与 SQLite 边界

数据分为三类，使用时不得混淆：

1. **官方文件确认**：可以从游戏官方静态文件直接读取，并在 SQLite 中保留来源文件、行键和
   SHA-256。
2. **项目金标准**：本文记录的伤害公式、乘区归类和默认值，由项目按已确认规则实现。
3. **待确认映射**：官方文件有原始数组，但没有足够证据把数组档位直接解释为具体角色或技能
   等级。此类数据只保存 `source_tier`，不猜测等级。

发行数据库为 `data/game_static.sqlite3`，战斗数据使用 schema v3 的下列表：

- `combat_level_curve`、`combat_level_curve_point`：倾陷确切等级曲线与环合官方档位。
- `reaction_definition`、`combat_effect_constant`：环合元素组合、默认伤害效果和固定常量。
- `skill_damage`、`skill_damage_rate`：技能伤害属性、固定暴击率、倾陷值以及攻击/生命/防御倍率。
- `enemy_combat_profile`、`enemy_element_resistance`：敌方防御、倾陷上限与分元素抗性。

计算 service 保持纯函数，不直接查询 SQLite；上层数据装配服务或 DAO 负责将静态数据转换成计算
输入。

## 直伤总公式

```text
最终直伤 = 技能伤害倍率
         × 倍率对应属性
         × 增伤区
         × 暴击区
         × 防御区
         × 抗性区
         × 易伤区
         × 所有独立乘区之积
```

技能伤害倍率来自已记录的技能伤害数据。倍率对应属性由技能记录指定，默认攻击力，也可以是
生命值或防御力；本次攻击的伤害属性由技能记录指定，例如暗属性或光属性。

## 基础属性区

```text
攻击力 = AtkBase × (1 + AtkUp) + AtkAdd
生命值 = HPBase × (1 + HPUp) + HPAdd
防御力 = DefBase × (1 + DefUp) + DefAdd
```

## 增伤区

```text
增伤区 = 1 + Σ(伤害提升)
```

伤害提升在此区相加，包括通用伤害增强、特定属性增强、特定技能增强、特定状态增强以及各类
Buff 提供的增伤。

## 易伤区

易伤是敌人身上的 Debuff，技能描述通常写为“受到的伤害提升”。它不计入我方的增伤区，而是
独立相加为一个乘区：

```text
易伤区 = 1 + Σ(敌人受到伤害提升)
```

## 暴击区

```text
暴击区 = 1 + 暴击率 × 暴击伤害
```

DOT 与部分环合伤害的固定 50% 暴击率规则按各自伤害类别处理。

## 防御区

默认场景为轨外之境：

```text
敌人防御 = (敌人等级 + 90) × (1 - 防御穿透) × (1 - 防御降低)
防御区 = (我方角色等级 + 100) / (敌人防御 + 我方角色等级 + 100)
```

大世界场景仅将敌人防御基数改为 `敌人等级 + 100`。后续其他场景应新增明确的基数规则。

## 抗性区

默认 Boss 基础抗性为 20%：

```text
X = Boss 基础抗性 - Boss 身上的减抗 Buff - 我方角色的抗性穿透 Buff

X >= 0：抗性区 = 1 - X
X < 0 ：抗性区 = 1 - X / 1.10
```

负抗性进入增伤区间；减抗来源包括技能和怪物弱点，抗性穿透来源包括武器和技能。

## 独立乘区

独立增伤不与增伤区相加，而是逐项相乘：

```text
独立乘区 = Π(1 + 每项独立增伤)
```

已知来源包括浸染增伤、早雾天赋的浊燃强化，以及浊燃状态下“目标每有一种持续伤害状态，受到
持续伤害提升 25%，最高提升 100%”。例如浸染 20%、早雾天赋 25%，独立乘区为
`1.20 × 1.25`。

## DOT 伤害

DOT（例如安魂曲的噩梦伤害）每秒结算一次。其倍率由技能数据库记录，基础乘区与直伤相同，
但暴击率恒定为 50%：

```text
DOT 单跳伤害 = 直伤总公式，其中暴击率固定为 0.50
```

DOT 有最大层数限制。当前采用按层独立结算的约定：结算时清空当前层数，对每一层使用该层的
剩余时长乘以单跳伤害，然后相加。

```text
DOT 结算总伤害 = Σ(第 i 层剩余时长 × DOT 单跳伤害)
```

当前按“每秒一跳”处理，因此剩余时长以秒为单位。该 DOT 结算约定可在获得更精确规则后替换。

## 环合基础规则

环合由角色打满环合条后，切换到相邻属性角色时触发切出角色的 QTE 与相应效果。环合归属角色
取参与者中 `环合强度 × 角色等级乘区` 较高者；该角色决定创生、浊燃等环合伤害的等级、环合
强度、防御与抗性相关乘区。

```text
环合强度乘区 = 1 + 环合强度 / 600
```

这里的 `/ 600` 表示“环合强度 / 6”是百分比增伤。覆纹、浸染使用的额外系数为：

```text
环合系数 = 24 × 环合强度 / (环合强度 + 180)
```

所有心灵伤害使用 100% 防御穿透，即防御区为 `1`；但仍计算心灵属性对应的抗性区，不能穿透
心灵防御抗性。

### 已记录的环合效果

- 创生（光+灵）：1 个创生株生成 5 个创生花；每 2 秒飞向目标爆炸，最多 3 株。伤害为等级乘区
  × 环合强度乘区 × 防御区 × 抗性区；80 级等级乘区为 9000；不吃覆纹增伤。
- 覆纹（灵+咒）：12 秒内灵/咒伤害结束时追加 `原伤害 × 20% × 环合系数` 的一次性伤害。
- 浊燃（咒+暗）：15 秒 DOT；80 级等级乘区为 2700，固定 50% 暴击，并使用环合归属角色的
  暴击伤害等参数。
- 黯星（暗+魂）：5 秒后结算心灵伤害；80 级等级乘区为 45000，不计算防御区。同一触发者不能
  重复叠加，不同触发者可叠加，最后一个黯星结束时统一结算总和。
- 浸染（魂+相）：12 秒内魂/相伤害的增伤区增加 `20% × 环合系数`。
- 延滞（相+光）：5 秒减攻减速。
- 盈蓄（光+灵+相）：创生花命中延滞目标时获得 10 点额外终结能量。
- 失谐（暗+魂+咒）：目标同时处于黯星与浊燃时直接扣除倾陷值；具体扣除量待补充。

## 倾陷伤害

```text
倾陷伤害 = 角色等级乘区 × 倾陷强度区 × 敌方倾陷上限区 × 防御区 × 抗性区
```

```text
倾陷强度区 = 1 + 队伍总倾陷强度 / 300 + Σ(倾陷伤害提高)
敌方倾陷上限区 = 敌方倾陷上限值 / 3
```

`UnbaldamagePara` 官方曲线明确记录了 1–80 级数据，其中 80 级角色等级乘区为 `3603`。轨外
之境 Boss 的敌方倾陷上限值默认 `50`；指定敌人时应优先读取其属性包中的 `UnbalMax`，只有
没有可用属性包时才使用默认值。倾陷伤害不使用攻击、
生命、防御倍率属性，也不使用增伤、易伤、暴击或独立增伤；防御区和抗性区沿用当前目标的对应
计算规则。

失谐只影响敌人进入倾陷的时间和加速退出倾陷的状态时序，不影响倾陷伤害，也不影响敌方倾陷
上限值的计算。

## 已确认映射与默认状态时序

### 技能倍率的 15 档

每个伤害 GE 在 `DT_SkillDamageData` 中保存攻击、生命、防御倍率数组。15 档数组按
`有效技能等级 - 1` 取值，超过数组末档时钳制到末档。有效技能等级不是直接相加觉醒等级：
当前确认规则为基础技能等级，加上“觉醒等级达到 3 时全技能 +1”。因此基础 10 级、三重觉醒
的技能取第 11 档（下标 10）；其他觉醒效果仍由调用方提供，避免未经确认的推断。

源数据路径：

`C:\softwares\codes\utils\异环\UnrealExporter\output_combat\HT\Content\DataTable\skill\DT_SkillDamageData.json`

SDK 结构为 `FSkillDamageExecutionData`，位于：

`C:\softwares\codes\utils\异环\5.6.1-0+UE5-HT(1)\5.6.1-0+UE5-HT\CppSDK\SDK\HTGame_structs.hpp`。

### 环合 16 档

确定采用每 5 级一档：

```text
档位 = floor((角色等级 - 1) / 5)
```

1–5 级为第 1 档，76–80 级为第 16 档，适用于创生、浊燃与黯星。

### 环合状态默认规则

- 除黯星外，重复施加暂按刷新持续时间、不创建新实例处理，标记为未实测。
- 黯星按触发者独立计时、独立爆炸；同一触发者再次施加时刷新自己的实例。
- 覆纹记录实际造成的伤害，到期追加 `实际伤害 × 20% × 环合系数`。
- 状态被提前移除时不立刻结算，除非技能专属效果明确要求结算。
- 不额外设定“同一帧”规则，事件按服务端接收先后处理。
- 失谐目前按 `敌方倾陷上限 × 15%` 扣除；固定值、等级系数、联机修正均记录为待定且暂不参与计算。
